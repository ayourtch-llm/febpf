Concurrency race detector

Runs several instances of the program concurrently against one shared map set, deterministically explores every interleaving, and reports data races — lost updates, stale read-modify-write. Every racing schedule is replayable. This is something kernel eBPF developers cannot do; febpf gets it from deterministic replay.
